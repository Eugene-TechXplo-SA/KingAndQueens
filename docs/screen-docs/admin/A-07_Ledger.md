# 台帳 / Ledger

## 画面概要


| 項目      | 内容                                        | English                                                      |
| ------- | ----------------------------------------- | ------------------------------------------------------------ |
| 画面名     | 台帳                                        | Ledger                                                       |
| 目的      | 全ユーザーの付与・減算・調整の履歴を閲覧し、「誰に・いつ・いくら」を説明可能にする | View all token credit/debit entries to ensure accountability |
| 主な利用ロール | ADMIN                                     |                                                              |
| 責務      | 台帳エントリ一覧表示、デルタ調整（手動補正）、CSV出力            |                                                              |
| やらないこと  | 付与計算・確定（A-01で実施）、出金操作（A-06で実施）            |                                                              |
| 画面構成    | 台帳一覧テーブル（フィルター・ソート付き）+ デルタ調整モーダル        |                                                              |

## アクション定義


| No  | トリガー           | 処理内容                          | 状態変更          | ログ有無 | 備考          |
| --- | -------------- | ----------------------------- | ------------- | ---- | ----------- |
| 1   | フィルター変更        | 台帳エントリをフィルタリング                | -             | -    |             |
| 2   | CSV出力ボタン       | 対象データのCSVを生成しダウンロード           | -             | -    |             |
| 3   | デルタ調整「適用」ボタン | LedgerEntry（ADJUSTMENT）を台帳に追記 | LedgerEntry作成 | 必須   | 理由必須（BR-04） |


## 状態遷移

該当なし（台帳は追記型。既存エントリの変更不可）

## ガード条件


| No  | ガード条件   | 内容                                 |
| --- | ------- | ---------------------------------- |
| 1   | Admin権限 | 台帳閲覧・調整はAdmin権限が必要                 |
| 2   | 調整理由必須  | デルタ調整時は理由の入力が必須                  |
| 3   | 台帳不変性   | 既存エントリの直接編集不可。修正は追記（ADJUSTMENT）で表現 |
