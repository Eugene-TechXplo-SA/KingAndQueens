# CSVインポート・台帳反映 / Trade Import (CSV Import)

## 画面概要


| 項目      | 内容                                         | English                                                                                 |
| ------- | ------------------------------------------ | --------------------------------------------------------------------------------------- |
| 画面名     | CSVインポート・台帳反映                               | Trade Import (CSV Import)                                                               |
| 目的      | ブローカーの取引履歴CSVを取込み、KQトークン付与量を計算・確定して台帳に反映する | Import broker trade history CSV, calculate KQ token rewards, and confirm ledger entries |
| 主な利用ロール | ADMIN                                      |                                                                                         |
| 責務      | CSV取込、原資プリフィル・確認、付与量計算プレビュー、付与確定・台帳反映      |                                                                                         |
| やらないこと  | ブローカー管理画面ログイン、CSV取得、実送金（これらはOff-System）    |                                                                                         |
| 画面構成    | KQレートカード + CSVアップロード領域 + プレビューテーブル + 確定ボタン |                                                                                         |
| パネル補足   | ブローカーからダウンロードしたCSV/Excelを取り込み、KQトークン付与を計算します |                                                                                         |


## 運用フロー

1. ブローカー管理画面からCSVをダウンロード（Off-System）
2. 本画面でCSVをアップロード → 自動検証・計算プレビュー
3. 内容を確認し「確定して台帳に反映」→ 台帳にCREDITエントリを追記

## アクション定義


| No  | トリガー             | 処理内容                                | 状態変更                                  | ログ有無 | 備考             |
| --- | ---------------- | ----------------------------------- | ------------------------------------- | ---- | -------------- |
| 1   | CSVファイルドロップ/選択   | ファイル解析・検証を開始。行ごとのバリデーション、重複検知を実施    | TradeImport: UPLOADED→VALIDATING      | -    | ブローカー自動判定      |
| 2   | 検証完了             | プレビューテーブルに計算結果を表示。原資をCSVから自動プリフィル   | TradeImport: VALIDATING→IMPORTED（成功時） | -    | 除外条件（BRK-01）適用 |
| 3   | 「確定して台帳に反映」ボタン押下 | 付与確定。LedgerEntry（CREDIT）を台帳に追記し残高反映 | DistributionRun作成→POSTED              | 必須   | 二重付与防止（BR-09）  |
| 4   | テンプレートCSVダウンロード  | CSVテンプレートファイルを生成しダウンロード             | -                                     | -    |                |


## 状態遷移


| 状態      | 説明                | 遷移先           | 備考       |
| ------- | ----------------- | ------------- | -------- |
| ファイル未選択 | 初期状態。ドロップゾーンのみ表示  | ファイル解析中       |          |
| ファイル解析中 | CSV解析・検証処理中       | プレビュー表示 / エラー |          |
| プレビュー表示 | 計算結果を確認可能。確定ボタン有効 | 確定済み          |          |
| 確定済み    | 台帳に反映完了           | ファイル未選択（次の取込） |          |
| エラー     | CSVフォーマット不備・検証失敗  | ファイル未選択       | エラー詳細を表示 |


## ガード条件


| No  | ガード条件       | 内容                        |
| --- | ----------- | ------------------------- |
| 1   | 取込停止スイッチOFF | 緊急停止（OPS-01）がOFFであること     |
| 2   | 対象ブローカーが有効  | broker.is_active = true   |
| 3   | 二重取込防止      | 同一バッチ/同一期間の重複チェック（BR-09）  |
| 4   | 原資入力済み      | 原資額が確定していること（BR-08）       |
| 5   | 必須マスター設定済み  | 配分比率・還元率・参考レート設定済み（BR-08） |
| 6   | 上限未超過       | 異常検知リミッター通過（BR-05）        |
