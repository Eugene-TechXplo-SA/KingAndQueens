# 出金管理 / Withdrawal Management

> **注記**: 出金管理は独立画面として実装されていません。
> 出金申請の管理機能は **A-02 承認タスク** の「出金申請」タブに統合されています。
> 出金ワークフロー（承認/却下・送金開始・TxHash登録）の詳細は [A-02_ApprovalTasks.md](A-02_ApprovalTasks.md) を参照してください。

## UI遷移フロー（参考）

```
承認タスク画面
  └─ 出金申請タブ（一覧テーブル）
       └─「操作」ボタン
            └─ 出金操作モーダル（詳細表示 + ステータス依存ボタン）
                 ├─ 承認 / 送金処理開始 / 送金完了 / 送金失敗
                 └─「編集」ボタン
                      └─ 出金申請編集モーダル（アセット/数量/送付先/ステータス/TxHash）
```

## 状態遷移（参考）

| 状態         | 説明                | 遷移先                             | 備考                 |
| ---------- | ----------------- | ------------------------------- | ------------------ |
| PENDING    | ユーザー申請済み、管理者の承認待ち | APPROVED / REJECTED / CANCELLED |                    |
| APPROVED   | 管理者承認済み。送金待ち      | PROCESSING                      |                    |
| PROCESSING | 送金実行中             | COMPLETED / FAILED              |                    |
| COMPLETED  | 送金完了              | -                               | TxHash必須、台帳DEBIT済み |
| REJECTED   | 管理者却下             | -                               | 却下理由保持             |
| CANCELLED  | ユーザーキャンセル         | -                               | ユーザー側操作            |
| FAILED     | 送金失敗              | -                               | 新規申請で対応            |

## ガード条件（参考）

| No  | ガード条件            | 内容                      |
| --- | ---------------- | ----------------------- |
| 1   | 出金停止OFF          | 緊急停止スイッチ（出金）がOFFであること   |
| 2   | ステータスがPENDING    | 承認/却下はPENDING状態のみ       |
| 3   | ステータスがAPPROVED   | 送金開始はAPPROVED状態のみ       |
| 4   | ステータスがPROCESSING | TxHash登録はPROCESSING状態のみ |
| 5   | 却下理由必須           | 却下時はreject_reasonの入力が必須 |
| 6   | TxHash一意         | 同一tx_hashの二重登録不可        |
