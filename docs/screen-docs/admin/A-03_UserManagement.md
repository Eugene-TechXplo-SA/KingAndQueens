# ユーザー管理 / User Management

## 画面概要


| 項目      | 内容                                                         | English                                                                   |
| ------- | ---------------------------------------------------------- | ------------------------------------------------------------------------- |
| 画面名     | ユーザー管理                                                     | User Management                                                           |
| 目的      | ユーザーの検索、詳細閲覧、情報編集、BAN/退会処理等を行う                             | Search users, view details, edit information, manage BAN/account deletion |
| 主な利用ロール | ADMIN                                                      |                                                                           |
| 責務      | ユーザー検索・一覧表示、詳細閲覧（口座/ウォレット/残高/ランク）、直接編集、BAN/解除、退会処理、KYC状態更新 |                                                                           |
| やらないこと  | 付与計算・確定（A-01で実施）、マスター設定（A-04で実施）                           |                                                                           |
| 画面構成    | ユーザー一覧テーブル + 詳細モーダル                                            |                                                                           |
| パネル補足   | ウォレット・ランク・残高の確認。BAN・トークン没収・退会処理                            |                                                                           |

## アクション定義


| No  | トリガー       | 処理内容                             | 状態変更                                 | ログ有無 | 備考          |
| --- | ---------- | -------------------------------- | ------------------------------------ | ---- | ----------- |
| 1   | 検索入力       | user_id/email/name等でユーザーを検索      | -                                    | -    | ページング対応     |
| 2   | 「詳細」ボタン    | ユーザー詳細モーダルを開く。口座/ウォレット/残高/履歴を取得  | -                                    | -    |             |
| 3   | メール直接編集    | ユーザーのメールアドレスを変更                  | User.email更新                         | 必須   | 例外対応用       |
| 4   | ランク変更（手動上書き） | ユーザーのランクを手動変更（通常は月次バッチで自動判定） | User.rank更新                          | 必須   | 例外対応用。通常は自動昇格・降格 |
| 5   | KYC状態更新    | KYC状態をAdmin操作で変更                 | User.kyc_status更新                    | 必須   |             |
| 6   | 「BAN」ボタン   | BAN理由入力後、ユーザーをBANNEDに更新          | User: ACTIVE→BANNED                  | 必須   | 理由必須        |
| 7   | 「BAN解除」ボタン | BANを解除しACTIVEに復帰                 | User: BANNED→ACTIVE                  | 必須   |             |
| 8   | 「没収」ボタン    | 没収量・理由入力後、台帳にDEBIT/CONFISCATEを追記 | LedgerEntry作成（DEBIT）                 | 必須   | 理由必須        |
| 9   | 「退会処理」ボタン  | 退会申請を確定し、WITHDRAWNに更新            | User: ACTIVE→WITHDRAWN               | 必須   | ログイン不可化     |


## 状態遷移


| 状態        | 説明         | 遷移先              | 備考     |
| --------- | ---------- | ---------------- | ------ |
| ACTIVE    | 通常利用可能     | BANNED / WITHDRAWN |        |
| BANNED    | 管理者による利用停止 | ACTIVE（BAN解除）    | 主要操作禁止 |
| WITHDRAWN | 退会確定       | -                | ログイン不可 |


## ガード条件


| No  | ガード条件   | 内容                               |
| --- | ------- | -------------------------------- |
| 1   | Admin権限 | ユーザー管理操作はAdmin権限が必要              |
| 2   | BAN理由必須 | BAN実行時はBAN理由の入力が必須               |
| 3   | 没収理由必須  | 没収実行時は没収理由と没収量の入力が必須             |
| 4   | 退会対象    | 退会処理はACTIVE状態のユーザーのみ             |
| 5   | 状態遷移整合  | WITHDRAWN→ACTIVEへの戻しは不可          |
