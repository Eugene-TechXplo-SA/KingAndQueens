# 目的・背景

## 1. 目的・背景

KING and QUEEN は、**海外FXで通常どおり取引するだけで、KQトークンが自動で蓄積される体験**を提供するサービスである。
従来の現金キャッシュバック型サービスに対して、**資産性・ゲーミフィケーション・運用拡張性**を加えた、次世代の還元体験を提供することを目的とする。

本サービスは、ユーザーにとっては「ポイントが増えるようなわかりやすさ」を持ちながら、出金時にはPolygon上でトークンとして受け取れるという、**Web2的UXとWeb3価値の橋渡し**を行う。

また、運営側にとっては、CSV取込・集計・計算・承認・台帳記録・出金承認といった業務を構造化し、**将来の自動化（スクレイピング / 自動送金）に耐える運用基盤**を作ることも重要な目的である。

## 1.1 ユーザー価値 / 事業価値

| 観点 | 内容 |
|------|------|
| ユーザー価値（資産性） | 現金ではなく、将来価値が上がる可能性のあるKQトークンを蓄積できる |
| ユーザー価値（簡単さ） | 口座開設→取引という既存行動の延長で還元を受けられる |
| ユーザー価値（透明性） | 取引量→計算→付与→出金の履歴が見える |
| 事業価値（差別化） | 現金キャッシュバック型サービスとの差別化が可能 |
| 事業価値（運用スケール） | CSV取込・承認ワークフロー・監査ログにより属人化を抑制できる |
| 事業価値（拡張性） | ブローカー追加・キャンペーン追加・付与ルール変更に対応しやすい |

## 1.2 ターゲットユーザー

### メインターゲット

- 海外FXユーザー（XM / FXGT 等）
- 20〜50代中心、トレード経験あり
- 現金キャッシュバックの仕組みは理解しているが、より"夢のある還元"を求める層

### サブターゲット

- クリプト経験者（ウォレット保有者）
- ポイ活ユーザー（数字が増える体験に反応する層）

## 1.3 プロダクト全体像（ユーザー視点）

ユーザー体験は以下の3本の流れで構成される。

1. **口座連携申請（還元対象化）**
   - 提携ブローカーで口座開設
   - MT4/MT5口座番号を申請
   - 管理者承認後、還元対象になる
2. **取引に応じたKQ付与（台帳反映）**
   - ブローカー報酬を原資に、運営がCSV取込・集計・計算・付与を実行
   - ユーザー残高としてKQが反映される
3. **出金申請〜ウォレット送付**
   - Izakaya Walletを連携
   - 出金申請
   - 管理者承認・手動送金（MVP）・TxHash記録
   - 出金履歴で確認
