# 用語定義（ドメイン用語）

> 本章は、PdM / FE / BE / デザイナー / 運用 / QA の間で、用語解釈のズレを防ぐための共通辞書である。
> ここで定義した用語を、仕様書・設計書・画面・API命名の基準とする

## 1.1 用語一覧（ドメイン辞書）

| 用語 | 定義 | 境界 / 責務 | 類似語との違い |
|------|------|-------------|---------------|
| KING and QUEEN（KQサービス） | 海外FX取引に応じてKQトークンを還元するサービス全体 | 口座連携、取引取込、付与、出金、管理運用を提供 | 単一機能ではなくサービス全体 |
| KQトークン | ユーザーへ還元される暗号資産 | 内部台帳で残高管理し、出金時にウォレットへ送付 | 単なるポイントではない（出金可能資産） |
| ポイント的UX | ユーザーにとって「増える数字」としてわかりやすく見せる体験。残高は通貨別に管理され、画面上に通貨別で表示する | ダッシュボード表示・履歴表示のUX方針 | KQトークンそのものの法的/技術的性質とは別 |
| ブローカー | 海外FX事業者（XM / FXGT 等） | 口座開設対象、取引履歴CSVの提供元、報酬支払元 | KQサービス運営とは別主体 |
| 口座連携申請 | ユーザーがMT4/MT5口座を還元対象として紐づける申請 | 承認/却下の審査対象 | 単純な口座登録ではない |
| 還元対象口座 | 管理者承認済みでKQ還元対象となる口座 | 取引量集計・付与計算の対象 | 未承認口座は対象外 |
| 取引履歴CSV | ブローカーから取得する取引データファイル | 集計・計算の入力元 | 直接API連携とは異なる（MVPは手動） |
| 取引データImport | CSVを内部台帳へ取り込む処理 | 集計・付与計算の前段 | 計算や付与確定そのものではない |
| 付与計算 | 原資（IB報酬）×還元率×ランク倍率をもとにKQ付与量を算出する処理 | プレビュー生成・付与確定前の計算 | 台帳記録（確定）とは別 |
| 付与実行 | 管理者が計算結果を確定し、付与台帳へ反映する操作 | ユーザー残高反映の確定トリガー | 計算プレビュー閲覧とは別 |
| 付与台帳 | 誰に・いつ・いくら付与したかの確定記録 | 残高反映根拠、監査・履歴参照の根拠 | ブロックチェーンTxそのものではない |
| 内部台帳（Ledger） | ユーザー、口座、取引、付与、出金の整合を管理するDB | 業務整合性の中心 | 市場/チェーンとは別レイヤー |
| Izakaya Wallet連携 | 出金先ウォレットをサービスに接続・確定する行為 | 公開アドレス等の取得・保存、受取先の確定 | 出金実行そのものではない |
| 出金申請 | ユーザーが内部残高のKQを外部ウォレットへ送付依頼すること | 審査対象・送金対象 | 単なる残高表示更新とは別 |
| TxHash | 送金トランザクションの識別子 | 送金結果の記録・履歴表示・監査根拠 | 内部の申請IDとは別 |
| 監査ログ | 管理者の重要操作履歴 | 不正/誤操作/障害時の追跡 | ユーザー向け履歴表示とは別 |
| 緊急停止スイッチ | CSV取込/付与/出金を停止する運用機能 | 異常時の被害拡大防止 | 通常の承認/却下とは別 |

### 追加用語

| 用語 | 定義 | 境界 / 責務 | 類似語との違い |
|------|------|-------------|---------------|
| 原資（IB報酬） | ブローカーから運営に支払われる報酬 | 還元率適用前の元の金額。付与計算の入力 | ユーザー還元原資（原資×還元率）とは別 |
| 還元率 | IB報酬のうちユーザーへ還元する割合 | マスター管理対象 | 配分比率（アセット間按分）とは別 |
| 配分比率 | ユーザー還元原資を複数アセットへ按分する比率 | マスター管理対象、合計100% | 還元率とは別レイヤー |
| DistributionRun（付与実行バッチ） | 付与計算結果を確定し台帳に書き込む実行単位 | DRAFT→CONFIRMED→POSTEDで台帳反映 | 計算プレビュー（RewardCalculationRun）とは別 |
| RewardCalculationRun（計算実行） | ロット集計結果から付与量を算出する処理の実行記録 | DRAFT→READY→付与実行(DistributionRun)へ | 付与確定（DistributionRun）とは別 |
| LedgerEntry（台帳エントリ） | 内部台帳の1行（CREDIT/DEBIT/ADJUST） | 残高増減の根拠記録、追記型・不変 | 残高そのものではない（集計結果が残高） |
| atomic（最小単位整数） | トークン数量の内部表現。浮動小数を避け最小単位の整数で保持する | API受け渡し、台帳記録の基本単位 | 表示上の小数値とは別（token_decimalsで変換） |
| TransferTx（送金トランザクション） | 出金の実送金結果を記録するエンティティ | TxHash、ネットワーク、result等を保持 | WithdrawalRequest（申請）とは別 |
| ランク / ランク倍率 | ユーザーに付与されるランクと、それに応じた還元率の倍率 | 付与計算時に倍率を適用（BR-10）。自動昇格・降格（月次バッチ）。閾値はマスター管理で変更可能 | 還元率そのものではない（還元率×ランク倍率で最終倍率） |
| KYC（本人確認） | 出金の前提となる本人確認ステータス | Phase1はAdmin手動でステータス管理（KYC-01） | 出金ゲートとしてのみ機能。審査フロー自体はPhase1では最小 |
| 除外条件 | ブローカー別に設定される報酬対象外取引の判定条件 | 最低取引時間等。マスター管理対象（BRK-01） | 異常検知（BR-05）とは別。正常な業務ルールとしての除外 |

## 1.2 用語運用ルール

### 1) UI表示名と内部用語の対応

- UI上では「ポイント的UX」で見せる場合があるが、内部仕様・設計上は **KQトークン / 付与 / 出金** の用語で統一する
- 「残高」は文脈に応じて **内部台帳残高（表示）** と **オンチェーン受取結果（出金完了）** を区別する

### 2) NG / 混同しやすい表現

- 「ポイント」と「トークン」を同義にしない
- 「申請受付」と「承認」を同義にしない
- 「計算完了」と「付与確定（台帳反映）」を同義にしない
- 「ウォレット連携」と「出金完了」を同義にしない
