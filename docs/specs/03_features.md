# 機能一覧

## 1.1 機能一覧（Phase1）

### 認証・ユーザー管理

| 機能ID | 機能名 | 目的 | 主体 | 優先度 |
|--------|--------|------|------|--------|
| AUTH-01 | ユーザー登録/ログイン | サービスを利用できる状態にする | User | Must |
| AUTH-02 | パスワード再設定/変更 | アカウント復旧・安全な運用 | User | Should |
| USR-01 | ユーザー詳細閲覧 | ユーザー情報（口座/ウォレット/残高/ランク）を確認 | Admin | Must |
| USR-02 | ユーザー情報の直接編集 | メール/ランク等の例外対応（監査必須） | Admin | Must |
| USR-03 | 強制退会（BAN）/解除 | 不正ユーザーを排除し安全運用 | Admin | Must |
| USR-04 | 退会申請 | ユーザーが退会を申請 | User | Should |
| USR-05 | 退会承認・退会処理 | 退会を確定しログイン不可にする | Admin | Must |
| KYC-01 | KYC状態管理（出金ゲート） | 出金に本人確認を必須化 | System/Admin | Must |

### 口座連携

| 機能ID | 機能名 | 目的 | 主体 | 優先度 |
|--------|--------|------|------|--------|
| ACC-01 | 口座連携申請作成 | 還元対象口座の申請を作成 | User | Must |
| ACC-02 | 口座連携申請 承認/却下（理由付き） | 還元対象口座を審査し確定 | Admin | Must |
| ACC-03 | 口座連携申請ステータス表示 | 申請状況をユーザーに提示 | User | Must |

### 取引取込・付与

| 機能ID | 機能名 | 目的 | 主体 | 優先度 |
|--------|--------|------|------|--------|
| TRD-01 | 取引履歴CSVインポート | 取引データを内部台帳へ取り込む | Admin | Must |
| TRD-02 | 取込バッチ管理/重複防止 | 二重取込を防ぎ追跡可能にする | Admin | Must |
| RWD-01 | 原資（IB報酬）入力 | 対象月のIB報酬額を入力し付与の前提を確定 | Admin | Must |
| RWD-02 | ロット集計・付与量計算（プレビュー） | ユーザー別付与量のプレビューを生成 | Admin | Must |
| RWD-03 | 付与実行・付与台帳記録 | 付与を確定し残高へ反映 | Admin | Must |
| RWD-04 | 付与取消（マイナス計上） | 誤付与を取り消し、監査可能にする | Admin | Must |
| RWD-05 | 端数処理 | 最小単位で計算を安定させる | System | Must |
| RWD-06 | 上限（リミッター）判定と手動承認 | 異常値をブロックし承認制に切替 | System/Admin | Must |
| RWD-07 | 除外条件適用（ブローカー別） | 報酬対象外取引を除外 | System/Admin | Must |
| RWD-08 | 損益非依存付与 | 取引の勝敗に関わらず付与（損益は判定に使用しない） | System | Must |

### ダッシュボード・履歴

| 機能ID | 機能名 | 目的 | 主体 | 優先度 |
|--------|--------|------|------|--------|
| DASH-01 | ダッシュボード表示 | 残高（通貨別）・履歴・次行動を提示 | User | Must |
| HIST-01 | 付与履歴表示 | アセット別付与履歴を確認 | User | Must |
| HIST-02 | 出金履歴表示 | 出金履歴/TxHash/ステータスを確認 | User | Must |

### ウォレット・出金

| 機能ID | 機能名 | 目的 | 主体 | 優先度 |
|--------|--------|------|------|--------|
| WAL-01 | Izakaya Wallet連携 | 出金先ウォレットを確定 | User | Must |
| WAL-02 | Wallet接続状態/受取アドレス表示 | Connected状態と0xアドレスを表示 | User | Must |
| WDR-01 | 出金申請作成（数量入力） | 内部残高のKQ出金申請を作成 | User | Must |
| WDR-02 | 出金申請 承認/却下（理由付き） | 出金申請を審査して可否を確定 | Admin | Must |
| WDR-03 | 手動送金・TxHash記録・ステータス更新 | 送金結果を記録し状態を更新 | Admin | Must |
| WDR-04 | 出金の最低条件チェック | 最低出金額・KYC等の前提判定 | System | Must |
| WDR-05 | 出金申請キャンセル | ユーザーがPENDING状態の自分の申請を取消 | User | Must |

### お知らせ（CMS）

| 機能ID | 機能名 | 目的 | 主体 | 優先度 |
|--------|--------|------|------|--------|
| CMS-01 | お知らせ作成/配信 | ユーザーへの告知を運用管理 | Admin | Must |
| CMS-02 | お知らせ閲覧 | ユーザーが告知を確認 | User | Must |

### マスター管理

| 機能ID | 機能名 | 目的 | 主体 | 優先度 |
|--------|--------|------|------|--------|
| MST-01 | マスター管理（統合） | ブローカー/還元率/上限等を運用管理 | Admin | Must |
| AST-01 | 付与アセットマスター管理 | 付与対象アセット（BTC/ETH/KQ等）の有効/無効管理 | Admin | Must |
| DST-01 | 配分比率設定管理 | アセット間の配分比率（合計100%）を管理 | Admin | Must |
| RNK-01 | ランクマスター管理 | ランク定義（名称・還元率倍率・昇格条件）を管理 | Admin | Must |
| CMP-01 | キャンペーン管理 | キャンペーン倍率（期間限定/入会特典等）を管理 | Admin | Must |
| BRK-01 | ブローカー設定管理 | メンテフラグ/除外条件/アフィリンク等を管理 | Admin | Must |
| PRC-01 | 価格参照設定 | 参考レート取得→管理者確定（付与計算に必須：BR-08） | Admin | Must |

### 台帳・監査・運用

| 機能ID | 機能名 | 目的 | 主体 | 優先度 |
|--------|--------|------|------|--------|
| LED-01 | 台帳（Ledger）閲覧 | 増減の根拠を追跡 | Admin | Must |
| LED-02 | 台帳delta調整（手動） | 緊急時の補正（理由+監査必須） | Admin | Must |
| AUD-01 | 監査ログ | 重要操作の追跡可能性を担保 | System/Admin | Must |
| OPS-01 | 緊急停止（取込/付与/出金） | 異常時に即停止 | Admin | Must |
| ALT-01 | 異常アラート（上限超/重複/急増） | 異常検知を可視化し事故を防ぐ | System/Admin | Must |
| SEC-01 | 重複/不正検知（警告） | 同一IP/同一Wallet等の重複を検知 | System/Admin | Should |

## 1.3 MST-01 マスター管理対象

| マスター | 管理内容 |
|----------|----------|
| 提携FX業者 | 初期3〜4社。IBログインリスト基準で登録 |
| 付与アセット（AST-01） | BTC/ETH/XRP/USDT/USDC/JPYR/JPYC/IZAKAYA/KQ |
| 配分比率（DST-01） | アセット間の配分比率（合計100%制約） |
| ランク（RNK-01） | ランク名・還元率倍率・昇格条件 |
| ブローカー別 還元率 | ブローカーごとの還元率設定 |
| ブローカー別 除外条件 | 最低取引時間等 |
| ブローカー別 メンテフラグ | 付与停止 |
| キャンペーン設定（CMP-01） | キャンペーン名・倍率・期間・対象条件 |
| 上限設定 | 異常検知用リミッター |
| 価格参照設定 | 参考レート（PRC-01） |
