# ロール / 権限

> 本章では、**業務として誰が何をできるか**を定義する。
> 実装上のRBAC詳細はFE/BE設計に委譲するが、業務上の責務分担は本章を基準とする。

## ロール一覧

### User（ユーザー）

海外FX取引を行い、還元を受ける主体。

| 区分 | できること |
|------|-----------|
| 操作 | 口座連携申請、ウォレット連携、出金申請、退会申請 |
| 閲覧（自分のみ） | 残高、ランク、付与履歴、出金履歴（TxHash含む）、お知らせ、KYC状態 |

### Admin（運営 / 管理者）

サービス運営主体。

| 区分 | できること |
|------|-----------|
| 審査 | 口座連携 承認/却下、出金 承認/却下 |
| 付与運用 | 原資入力、CSV取込、集計、付与計算/確定、付与取消 |
| 出金運用 | 手動送金（システム外）、TxHash記録 |
| マスター | ブローカー、還元率、配分比率、ランク、上限、キャンペーン、付与アセット、価格参照 |
| ユーザー管理 | 検索、詳細閲覧、情報編集、ランク変更、BAN/解除、退会処理、KYC状態更新 |
| 台帳・監査 | 台帳閲覧、delta調整、監査ログ閲覧 |
| 運用制御 | 緊急停止（取込/付与/出金）、アラート確認/対応 |

> MVPでは **Adminが全権限** を持つ。Viewer / Approver 等への分離は Phase2。

## 権限制約

### User

- **自分の**申請・残高・履歴のみ閲覧/操作可能
- 他ユーザーの情報にアクセス不可
- 管理者操作（承認/却下、付与確定、マスター変更等）は不可
- 管理画面へのアクセスは不可

### Admin

- 全件閲覧・審査・運用操作が可能
- 状態遷移に反する操作は実行不可（例：`COMPLETED` → `APPROVED` への戻し不可）
- 緊急停止ON中は該当機能が実行不可

## Phase2拡張

- Admin権限の分割（Viewer / Operator / Approver）
- 二重承認（4-eyes）導入（付与確定・残高調整・BAN等）
- Off-System作業の自動化
