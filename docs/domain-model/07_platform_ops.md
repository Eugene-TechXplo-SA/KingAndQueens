# Platform Ops

> Notion Source: https://www.notion.so/31b541c6043480a9b9dae4f5d1cfcef0

## 概要

各ドメインの処理を横断的に制御・監視する運用機能をまとめたドメイン。
緊急停止スイッチ、参考レート設定、上限リミッター、監査ログ、アラートを管理する。

---

## ユースケース

### A-02 承認タスク一覧閲覧
- **目的：** 未承認の口座連携/出金申請を確認
- **前提：** 管理者ログイン済み
- **勝利フロー**
  1. 管理者が「承認タスク一覧」を開く
  2. サーバーが PENDING の口座連携申請（AccountLinkRequest）を取得
  3. サーバーが PENDING の出金申請（Withdrawal）を取得
  4. ページング・ソート条件に従って一覧を返す
- **例外フロー**
  - 権限不足：403
- **結果：** PENDING一覧が取得できる（口座連携/出金）

---

### P-01 監査ログの記録
- **目的：** 重要操作の追跡（付与確定、設定変更、BAN等）
- **前提：** 管理者操作（Admin認証済み）またはシステム操作としてログ対象
- **勝利フロー**
  1. 重要操作（例：承認/却下、付与確定、設定変更、BAN、Tx登録）が実行される
  2. サーバーが「誰が（admin_id）・いつ・何を（action）・対象（entity_type/id）・結果（success/failure）」を組み立てる
  3. AuditLog を追記（作成）
  4. （推奨）`X-Request-Id` をログに紐づける
- **例外フロー**
  - ログ書込失敗：原則は操作全体を失敗扱い（500）にするか、ログだけ後追いするか方針固定
- **結果：** 監査ログが作成され、重要操作が追跡可能になる

---

### P-02 参考レート設定（アセット別参考レート）
- **目的：** 付与計算のアセット数量変換およびダッシュボードの円換算表示に使うレートを管理者が確定
- **前提：** Admin認証済み、設定変更権限あり
- **勝利フロー**
  1. 管理者が「参考レート設定」画面で `rate` と適用日（当日）を入力して送信
  2. サーバーが入力形式・範囲を検証（負数不可、桁数上限など）
  3. TokenRate を更新（履歴を持つ場合は新規作成して有効化）
  4. 監査ログに「レート設定」を記録
  5. 現在有効なレートを返す
- **例外フロー**
  - 入力不正：422
  - 権限不足：403
  - 競合（同日レートを二重に確定しない方針の場合）：409
- **結果：** TokenRate が更新/作成され、円換算表示や計算の前提として参照可能になる

---

### P-03 上限/リミッター設定
- **目的：** 異常検知・安全装置（付与上限など）
- **前提：** Admin認証済み、設定変更権限あり
- **勝利フロー**
  1. 管理者が日次/月次などの上限値（LimitRule）を入力して送信
  2. サーバーが入力形式・範囲を検証（負数不可、上限の整合など）
  3. LimitRule を更新（履歴を持つ場合は新規作成して有効化）
  4. 監査ログに「上限設定」を記録
  5. 現在有効な上限設定を返す
- **例外フロー**
  - 入力不正：422
  - 権限不足：403
- **結果：** LimitRule が更新され、取込/計算/付与の処理で上限判定に利用される

---

### P-04 緊急停止スイッチ
- **目的：** 取込停止/付与停止/出金停止/口座連携受付停止を即時反映
- **前提：** Admin認証済み、緊急停止権限あり
- **勝利フロー**
  1. 管理者が緊急停止スイッチを切り替える（例：`import_enabled=false`）
  2. サーバーが EmergencySwitch を更新
  3. 監査ログに「緊急停止変更」を記録
  4. 対象ドメインの処理は即時ブロック（停止中は503を返す、全ドメイン統一）
     - `import_enabled`：CSV取込停止
     - `distribution_enabled`：付与確定（DRAFT→CONFIRMED）をブロック
     - `withdrawal_enabled`：新規出金申請・承認をブロック
     - `account_link_enabled`：口座連携申請をブロック
  5. 現在のスイッチ状態を返す
- **例外フロー**
  - 権限不足：403
- **結果：** EmergencySwitch が更新され、対象処理が即時停止/再開できる

---

### P-05 アラート（任意）
- **目的：** 上限超過、重複、急増などを運営に提示
- **前提：** アラート機能を採用する（Phase1でどこまでやるか要固定）
- **勝利フロー（生成）**
  1. 取込・計算・付与・承認の処理中に異常条件を検知（例：上限超、重複、急増）
  2. サーバーが Alert を作成（severity、type、context情報を保存）
  3. 管理画面で一覧表示できるようにする
  4. （任意）通知やバッジに連携
- **勝利フロー（運用）**
  1. 管理者がアラート一覧を確認
  2. 必要に応じて ACK（確認済み）/ CLOSE（対応完了）に更新
  3. 操作を監査ログに記録
- **例外フロー**
  - 権限不足：403
- **結果：** Alert が作成され、運営が異常に気づいて対応できる

## 緊急停止スイッチ（EmergencySwitch）

各ドメインの処理を即時停止/再開するためのスイッチ。

### スイッチ一覧

| スイッチ | 影響範囲 | 停止時の挙動 |
|---------|---------|-------------|
| `import_enabled` | Trade Import | CSV取込APIが503を返す |
| `distribution_enabled` | Rewards | 付与確定（DRAFT→CONFIRMED）をブロック |
| `withdrawal_enabled` | Withdrawals | 新規出金申請・承認をブロック（503） |
| `account_link_enabled` | Broker Integration | 口座連携申請をブロック（503） |

- 各スイッチは独立してON/OFF可能
- 停止中の処理は `503` を返す（全ドメイン統一）
- 切替は監査ログ対象

---

## 参考レート設定（TokenRate）

付与計算のアセット数量変換およびダッシュボードの円換算表示に使用するレート。

### 管理ルール
- Adminが管理画面からアセット別のレートを設定
- 適用日（当日）を指定して登録
- 履歴を保持（過去のレートを参照可能）
- 同日に同一アセットのレートを二重に確定しない
- 設定変更は監査ログ対象

---

## 上限リミッター（LimitRule）

異常検知用の安全装置。ビジネス上の上限は設けないが、バグ/不正/急増に備えた日次・月次の上限を設定する。

### 管理ルール
- Adminが管理画面から日次/月次の上限値を設定
- 上限以内：自動付与（通常処理）
- 上限超過：**手動承認が必要**（管理画面で確認）
- 設定変更は監査ログ対象

---

## 監査ログ（AuditLog）（BR-04）

重要操作を「誰が・いつ・何を・どの対象に・なぜ・どう変えたか」で追跡する。

### 記録項目

| 項目 | 内容 |
|------|------|
| 実行者 | admin_id |
| 実行日時 | timestamp |
| 操作種別 | action |
| 対象 | entity_type / entity_id |
| 差分 | 変更前 / 変更後 |
| 理由 | 自由記述 or 理由コード |

### 対象操作一覧

| カテゴリ | 操作 |
|---------|------|
| 口座連携 | 承認/却下 |
| 取引取込 | CSV取込確定 |
| 付与 | 原資入力、付与確定、付与取消、台帳delta調整 |
| 配分・ランク | 配分比率変更、ランク設定変更、還元率変更 |
| マスター | 還元率、上限、キャンペーン、価格参照、付与アセット設定変更 |
| 出金 | 承認/却下、TxHash記録、ステータス更新 |
| 運用 | 緊急停止ON/OFF |
| ユーザー管理 | 情報直接編集、BAN/解除、退会処理、KYC状態更新 |

### 不変ルール
- 監査ログは **追記のみ**（削除・編集不可）
- ログ書込失敗時は原則として操作全体を失敗扱いにする

---

## アラート（Alert）（BR-05）

異常検知時に管理者へ提示するアラート。

### 検知対象
- 同一データの重複取込疑い
- 集計ロットの急増
- 異常な付与量（上限超過）
- 異常な出金申請

### アラート運用
- 取込・計算・付与・承認の処理中に異常条件を検知 → Alert を生成
- 管理画面でアラート一覧を確認
- 管理者が `ACK`（確認済み）/ `CLOSE`（対応完了）に更新
- 操作は監査ログに記録

---

## 処理フロー

### 緊急停止の切替
1. Adminがスイッチを切り替える（例：`withdrawal_enabled = false`）
2. EmergencySwitch を更新
3. 監査ログに記録
4. 対象ドメインの処理が即時ブロック/再開される

### 参考レート設定
1. Adminがアセット別のレートと適用日を入力
2. 入力形式・範囲を検証（負数不可、桁数制限）
3. 同日・同一アセットの重複チェック
4. TokenRate を作成/更新
5. 監査ログに記録

### 上限リミッター設定
1. Adminが日次/月次の上限値を入力
2. 入力形式・範囲を検証
3. LimitRule を更新
4. 監査ログに記録

---

## 管理画面（見せたいもの）

### 緊急停止
- 各スイッチの現在状態（ON/OFF）
- ワンクリックでの切替操作
- 切替履歴（監査ログ連携）

### 参考レート
- アセット別の現在レート
- レート設定・変更フォーム
- 設定履歴

### 上限リミッター
- 現在の上限値（日次/月次）
- 上限変更フォーム
- 上限超過時の手動承認キュー

### 監査ログ
- 全操作の検索・フィルタ（期間/操作種別/実行者/対象）
- 詳細表示（差分/理由）

### アラート
- アラート一覧（severity / type / 状態 / 日時）
- ACK / CLOSE 操作
- コンテキスト情報の表示

## 通知

| イベント | ユーザー通知 |
|---------|-------------|
| 緊急停止ON/OFF | なし（管理者向け） |
| レート設定変更 | なし（管理者向け） |
| 上限超過アラート | なし（管理者向けアラート） |

---

## 不変条件（Invariants）

1. **監査ログ不変：** 監査ログは追記のみ、削除・編集不可
2. **停止時の一貫性：** 緊急停止ON時は対象ドメインの全APIが503を返す（統一挙動）
3. **レート一意性：** 同日・同一アセットのレートは1件のみ
4. **上限超過時の安全性：** 上限超過は自動確定せず手動承認必須

---

## 決定済み事項

- ビジネス上の付与上限は設けない（異常検知用リミッターのみ）
- 緊急停止は各機能（取込/付与/出金/口座連携）を独立して制御可能
- 停止中のAPIは503を返す（全ドメイン統一）
- 監査ログは追記型（削除不可）
